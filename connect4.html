  • Connect 4 Game

"Programmatically create a game of Connect 4™ with one AI-controlled opponent and a human-controlled player."

Note: Although I did not have time to finish the program, I did create a design doc for if I wanted to implement it later.

General Game Rules:

  • Connect 4 is a 7 slot wide, 6 slot high game
  • There are 2 players, one red, one black
  • Players take turns placing one in a slot until they have 4 adjacent pieces in a “row” either horizontally, verically or diagonally

Basic Structure of Program (Assume no AI for the moment):

  • Game board starts out empty
  • In a loop until game has been won or no more moves available:
    • Switch players (on first move this will set to player 1)
    • Wait for current player to place a piece
      • If player clicks somewhere, check if it is a legal move, if so, a piece is placed
  • When game is over the game will prompt for another game
    • If yes restart game
    • If no, exit program (or return to a title screen or do whatever)

The GUI (Brainstorming, some of this may not be implemented):

  • Should start with a white background, a simple window with OS look-and-feel
  • Yellow box with circular holes
  • Current player's turn above in large text
  • Sidebar on right with new game and help buttons
  • When hovering above the game board, make the piece appear (only if it's the user's turn)
  • If the user clicks and they are above a column and it is legal they can move
  • Notification box for game-over, prompting user for a new game (or just an okay box)

Logic-side:

External loop

  • Set board to empty
  • Setup all game flags/boolean values to false
  • Run main game loop

Main game loop (Pure logic no GUI)

  • Get move
  • Check for game over
    • If so break from game loop
  • Change players

Conditions for game over:

  • Board is full (no moves available, draw)
  • Player 1 or 2 wins
    • 3 Possible ways: Diagonal, Vertical, Horizontal

Making a move:

  • Validate move
  • Place piece
  • Check if the board is full, if so set a boolean value for draw
  • Check if player wins, if so set a boolen value for player win

AI Design:

  • Search for best available move (based upon priority):
    1. If AI has 3 in a row, place fourth
    2. If opponent has 3 in a row attempt to block
      • If there are multiple 3 in a row, then block first found
    3. If opponent has 2 in a row and 1 on other side, block it
    4. Search for longest consecutive sequence of AI (1 or 2) and place
    5. If AI has no sequence, place to block player's 2 in a row
  • Issues with this method:
    • How can I set the AI to place 4 in a row on both sides? (with a hole to fill)
    • How will AI keep track of diagonals and stacking strategy?
    • Will this be competitive to user?
